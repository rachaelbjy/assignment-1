Technical Enhancements

A showcase of advanced HTML5 and pure CSS techniques used to elevate the user experience across the website.

1. "Maximum Chaos" Emoji Overlay

Where to find it: Global Page Load (Themed differently per page).

How we did it: We created a full-screen overlay using position: fixed. To make sure it doesn't block users, we added pointer-events: none. The emojis fly across the screen using @keyframes and viewport units. We kept emojis in the HTML so we could swap them per page without changing CSS.

2. Stationary Typewriter Animation

Where to find it: Hero Headers (H1).

How we did it: To make the main heading look like it is being typed out, we used @keyframes and clip-path: inset(). This creates an invisible box that slowly pulls back to reveal the text letter-by-letter instead of sliding into view.

3. Automatic System Dark Mode Detection

Where to find it: Global (Across the entire website).

How we did it: We implemented an automatic dark theme using the advanced CSS media query @media (prefers-color-scheme: dark). The website automatically detects if the user's Mac or device is set to Dark Mode and seamlessly swaps the backgrounds, form inputs, and text colors to a dark slate palette. This provides a premium, highly accessible user experience without writing a single line of JavaScript.

4. Smart "Back to Top" Button

Where to find it: Global (Bottom right of every page).

How we did it: We implemented a smooth-scroll navigation button that only appears after the user leaves the top of the page. We used the modern animation-timeline: scroll() property to handle the visibility automatically. This ensures the UI stays clean on landing while providing easy navigation for long product pages.

5. Cascade Grid Loading Animation

Where to find it: Home Page Catalogues and Product Grids.

How we did it: To create a premium, staggered entrance effect without relying on JavaScript scroll listeners, we utilized pure CSS @keyframes. We designed a smooth fade-up animation and applied it to the grid items. By leveraging the advanced :nth-child() structural pseudo-class, we applied sequential animation-delay values to each consecutive card, causing them to cascade onto the screen beautifully upon page load.

6. Global Smooth Page Transitions

Where to find it: Global (Every page load).

How we did it: To eliminate the jarring, instant visual flash that normally occurs when navigating between standard HTML pages, we implemented a global fade-in effect. By applying a custom CSS @keyframes animation directly to the body element, every page smoothly transitions into view upon load. The forwards animation fill mode ensures the document remains fully visible after the animation completes, providing a seamless, app-like browsing experience.
